# Introduction

Welcome to Spartan Vue! This comprehensive guide will help you get started with our Vue.js component library and understand how to integrate it into your projects effectively.

## What is Spartan Vue?

Spartan Vue is a modern, production-ready component library built specifically for Vue.js applications. It provides developers with a comprehensive collection of reusable UI components that prioritize accessibility, performance, and developer experience.

## Design Philosophy

Spartan is designed to be the foundation for all PlacetoPay applications, built on four core principles:

**PlacetoPay Design Consistency**: Maintains the visual identity and design standards of PlacetoPay while providing flexible variants that allow clients to customize and identify with their brand

**Utility & Reusability First**: Provides ready-to-use components so developers can focus on business logic instead of UI implementation when building PlacetoPay applications

**Scalable Architecture**: Built to grow with your project needs, from small integrations to large enterprise applications, without compromising performance or maintainability

**Mobile-First Responsive**: Every component is designed with mobile devices as the primary consideration, ensuring optimal experience across all screen sizes and devices

### KEY BENEFITS

- üöÄ Rapid Development: Skip the tedious work of building common UI components from scratch. Our library provides battle-tested components that you can use immediately.
- üé® Professional Design: Every component is carefully crafted with modern design principles.
- ‚ôø Built-in Accessibility: All components include proper ARIA attributes, keyboard navigation, and screen reader support.
- üîß TypeScript Ready: Full TypeScript support with comprehensive type definitions.
- üì± Mobile-First: Responsive design is built into every component.

## Who Should Use Spartan Vue?

# Migration Guide: Spartan Vue 2.x to 3.x

Migrating from Spartan Vue 2.x (Tailwind CSS 3) to Spartan Vue 3.x (Tailwind CSS 4).

## Quick Overview

Spartan Vue 3.x introduces full support for Tailwind CSS 4 with a CSS-native configuration approach.

### Migration Steps Summary

1. **Remove** Spartan Vue plugin from `tailwind.config.js`
2. **Run** `npx @tailwindcss/upgrade` to migrate to Tailwind 4
3. **Remove** PostCSS configuration (if applicable)
4. **Install** `@tailwindcss/vite` plugin (if using Vite)
5. **Update** to Spartan Vue 3.x: `npm install @placetopay/spartan-vue@latest`
6. **Move** CSS imports from JavaScript to your main CSS file
7. **Update** color imports API (if using Tailwind colors in JS)
8. **Test** your application

## Key Changes (Breaking Changes)

- No more JavaScript `tailwind.config.js` - now uses CSS-based configuration
- CSS imports moved from JavaScript to CSS files using `@import`
- Tailwind color imports API changed from named exports to default export
- Single `styles.css` import replaces multiple imports

## Resources

- Full Migration Guide: https://github.com/placetopay-org/spartan-vue/blob/main/MIGRATION.md
- Tailwind CSS v4 Upgrade Guide: https://tailwindcss.com/docs/upgrade-guide
- Spartan Vue Documentation: https://develop--646e732a14dfaa707ad59b33.chromatic.com/
- Report Issues: https://github.com/placetopay-org/spartan-vue/issues

type TBindings = {
    check?: string[];
    prop?: Record<string, string | boolean | undefined>;
    emit?: string[];
    custom?: Record<'icon', boolean>;
};

const getBinding = (binding: string, map?: string) =>
    map ? `${map}Prop${binding.charAt(0).toUpperCase() + binding.slice(1)}` : binding;

export const buildSourceBinding = (bindings: TBindings, map?: string) => (args: any) => {
    let result = '';

    if (bindings.prop) {
        Object.keys(bindings.prop).forEach((prop) => {
            if (args[prop] !== undefined && args[prop] !== '' && args[prop] !== bindings.prop![prop])
                result += `${prop}="${args[prop]}" `;
        });
    }

    if (bindings.custom?.icon) {
        if (args[getBinding('icon', map)]) result += `:icon="${args[getBinding('icon', map)]}" `;
    }

    if (bindings.check) {
        bindings.check.forEach((binding) => {
            if (args[getBinding(binding, map)]) result += `${binding} `;
        });
    }

    if (bindings.emit) {
        bindings.emit.forEach((emit) => {
            if (args[emit]) result += `@${emit}="${args[emit].replace(/ /g, '')}" `;
        });
    }

    return result.trim();
};
