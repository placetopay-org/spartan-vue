## When to use it

- Physical addresses (country â†’ state â†’ city â†’ neighborhood) in profiles or checkouts.
- Business hierarchies: sector â†’ industry â†’ sub-industry.
- Any scenario where one selection drastically narrows the option set of the next step.

## Design decisions

### Visual attachment

**No attachment.** The selectors stack vertically with `space-y-3`; each one has its own `SLabel`. Joining them with `rounded` would be misleading because these are sequential decisions, not a single value. The vertical layout makes the order explicit.

### v-model shape

Three independent refs (`country`, `state`, `city`). Each `SSelector` owns its state; the link between them is derivation logic in `computed` and reset logic in `watch`.

At submit time, assemble the object `{ country, state, city }`. Treat the value as "complete" only when all three are defined.

### Error coordination

- `error` on the last empty selector if the form was submitted and the chain is incomplete.
- Do not propagate the error upward: if only the city is missing, the country selector has no reason to turn red. The dependency chain is vertical, not horizontal.

### Cascading disabled

- `state` starts `disabled` until `country` is set.
- `city` starts `disabled` until `state` is set.
- When a parent selector changes, **reset** every dependent child. This prevents impossible combinations like "Colombia + Jalisco".

### Label and error message

Each selector carries its own `SLabel` ("Country", "State", "City") because they are distinct questions. The error message lives next to the affected selector, not grouped at the bottom.

### Accessibility

- Each `SSelector` should carry an `id` linked to its `SLabel` via `for`.
- The `disabled` state is announced automatically by the component.
- Natural tab order: country â†’ state â†’ city.

## Variants

- **Async loading**: replace the inline structures with `fetch` calls inside `watch`. Show `loading` on the dependent selector while the response is in flight. Useful when the catalog is large or changes often.
- **Per-level search**: enable `search` on selectors with more than ~8 options (typically city). Avoid enabling it on country if the set is short.
- **Preloading existing values**: when editing an address, hydrate the three refs in order (`country` â†’ `state` â†’ `city`) while respecting the dependencies â€” a single bulk assignment can leave intermediate refs empty because of the reset `watch`.
