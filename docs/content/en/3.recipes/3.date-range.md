Capture a "from / to" time range with two independent `SInputDate` components, one per end of the range.

::component-preview{file="recipes/date-range"}
::

## When to use it

- Report filters, dashboards, and audit tools where the user picks a period.
- Booking forms (check-in / check-out) where each date opens its own calendar.
- Cases where the user should be able to edit each end without losing the other — a known friction with `selectionMode="range"` when they only want to change "to".

## Design decisions

### Visual attachment

**The two inputs are NOT attached.** `SInputDate` does not expose `rounded`, and each end has its own calendar popover. Place them side by side with `grid grid-cols-2 gap-3` or `flex gap-3`. A small caption ("From" / "To") above each clarifies its role without needing icons.

### v-model shape

**Recommended**: two independent refs `from: Date | null` and `to: Date | null`. Each `SInputDate` is fully autonomous and validated on its own.

Alternative: an object `{ from, to }` if your back-end expects that literal shape. Building the object in a `computed` at submit time is usually cleaner than keeping it as reactive state.

### Error coordination

When `to < from`, flag `error` on **both** inputs. This is the most common case; the combo rule ("both red when the relation fails") applies here too, even though the inputs are not physically attached.

For per-end errors (e.g., "from" alone is outside an allowed window), flag `error` only on the affected end.

### Label and error message

One `SLabel` above the combo with the range's name ("Date range", "Report period"). Below the pair, a single paragraph describes the error in range terms ("End date cannot be earlier than start date") rather than repeating it per input.

### Accessibility

- Each `SInputDate` carries its own `id` (`range-from`, `range-to`). The "From"/"To" captions can be decorative `<span>`s or `<label for>` if you need the explicit link.
- If you show an error, add `aria-describedby` on both inputs pointing at the shared error paragraph.
- Natural tab order: `from` → `to`. Respect the visual order.

## Variants

- **Single input with `selectionMode="range"`**: native `SInputDate` captures the range in one input with a single calendar. More compact, avoids the cross-validation. Use that form when space is tight (table cells, mobile filters); use this recipe when the user needs to see and edit each end separately.
- **With quick presets**: add "Today", "This week", "This month" buttons above or next to the pair that write both dates at once. Nice for report filters.
- **Min / max date**: each `SInputDate` accepts its own constraints (`minDate`, `maxDate`). Use the other end's value as a dynamic bound (`minDate: from`) to prevent inversions from the calendar.
