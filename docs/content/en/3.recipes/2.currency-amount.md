Pick the currency and capture the amount in a single pill where the number formatting responds to the active currency.

::component-preview{file="recipes/currency-amount"}
::

## When to use it

- Payment forms where the user can pay in multiple currencies (USD, COP, EUR, etc.).
- Admin tools that create quotes or invoices across currencies.
- Any scenario where `SInputAmount`'s native `rightOptions` is not enough: you need search, grouping by region, or options with icon or flag.

## Design decisions

### Visual attachment

`rounded="left"` on the currency `SSelector` and `rounded="right"` on the `SInputAmount`. `SInputAmount` internally wraps `SInput`, so the same height and outline rules apply. Use `flex items-stretch` so the text baseline stays aligned between the two.

### v-model shape

**Recommended**: two refs (`currency` + `amount`), not an object. The amount is emitted as `number | null` (driven by `vue-currency-input`) and the currency as an ISO 4217 code â€” wrapping them in an object would force serialize/deserialize cycles on every change. If your back-end expects `{ currency, amount }`, assemble it at submit time.

The `currency` prop of `SInputAmount` MUST be bound to the currency ref so formatting re-runs when it changes.

### Error coordination

- `error` on both when the amount violates a currency-specific min/max.
- `error` on just the `SInputAmount` when the format is invalid (rare; the mask usually prevents it) and the currency is correct.

### Label and error message

One `SLabel` "Payment amount" above the pill. Below, a single message describes the error in combo terms ("Minimum amount in USD is 10") rather than separate messages per child.

### Accessibility

- `id` on the `SInputAmount` connected to `SLabel` via `for`.
- `aria-describedby` pointing at the error paragraph when it appears.
- Tab order: `SSelector` â†’ `SInputAmount`. The currency list must be keyboard-navigable (`SSelector` already is).

## Variants

- **With a flag per currency**: `SSelector`'s `option` slot accepts an `<img>` or a flag emoji next to the code.
- **Currencies grouped by region**: use `optionGroupLabel` and `optionGroupItems` to group (LATAM, Europe, Asia).
- **Fixed currency**: if the form uses one currency, use `SInputAmount` with the `symbol` prop instead of this recipe.
