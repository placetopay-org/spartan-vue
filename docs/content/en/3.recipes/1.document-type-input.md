Capture an identification type (ID, passport, tax ID, etc.) and its number as if they were a single form field.

## When to use it

- Sign-up or profile forms where the type conditions the validation of the number.
- Any "category + value" combo where the selector picks a small set of options and the input captures the main data (tax code + base amount, product code + quantity, etc.).
- When the document-type options are objects (with `label` and `code`) and you need the full power of `SSelector`: search, grouping, loading, clearable.

## Design decisions

### Visual attachment

Attachment is purely typographic: `rounded="left"` on the `SSelector` and `rounded="right"` on the `SInput`. Both share the same height (`h-9`) because they use the same `inputStyle.root` token. The wrapper is `flex items-stretch` so the bottom edges stay aligned even when the browser applies `autofill`.

### v-model shape

**Recommended**: a single object `{ type, number }` bound as the combo's `v-model`. The type and the number are one data unit and rarely travel separately to the back-end. Binding them as a single object makes that inseparability explicit.

Acceptable alternative: two independent refs (`type`, `number`) when the enclosing form already works with flat fields. In that case, combine them in a `submit` function before dispatching.

### Error coordination

If validation of the combo fails (for example, an invalid number for the chosen type), pass `error` to **both** components. The rule: the combo reads as one pill, so a "half-red" pill feels like a render bug. The consumer keeps a single `hasError` computed and forwards it to both children.

### Label and error message

Use a single `SLabel` above the combo (not one per component) and a single error message below. That respects that the combo asks one question ("what is your document?"), not two.

### Accessibility

- The `SInput` carries the `id` referenced by the `SLabel` via `for`. The `SSelector` inherits its label by proximity and visual relation.
- When the combo shows an error, add `aria-describedby` on the `SInput` pointing at the error paragraph's id.

## Variants

- **With search**: if the type list exceeds ~8 entries (e.g., enterprises with many tax categories), enable `search` on `SSelector`.
- **With per-type icons**: use `SSelector`'s `option` slot to render an icon or flag next to each entry. The closed button stays compact when `optionLabel` points to the short code.
- **Async types**: fetch the types in `onMounted` and show `loading` on `SSelector` while data is in flight.
