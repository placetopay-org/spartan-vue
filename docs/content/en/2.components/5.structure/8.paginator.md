# Usage

The `SPaginator` component renders pagination controls. Provide `page`, `size`, and `total` to calculate page numbers automatically.

::component-preview{file="SPaginator/basic"}
::

## Page Sizes

Use the `pageSizes` prop to display a dropdown that lets users change the number of results per page.

::component-preview{file="SPaginator/page-sizes"}
::

## Hide Numbers

Set `hideNumbers` to `true` to display only the previous and next navigation arrows without page numbers.

::component-preview{file="SPaginator/hide-numbers"}
::

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `page` | `number` | - | The current page number (v-model:page) |
| `size` | `number` | - | The number of items per page (v-model:size) |
| `total` | `number` | - | The total number of items |
| `count` | `number` | - | The total number of pages (overrides total/size calculation) |
| `pageSizes` | `number[]` | - | Array of page size options for the dropdown |
| `paginatorSize` | `string` | - | Number of page buttons to show around the current page |
| `hideNumbers` | `boolean` | `false` | Hides page number buttons |
| `hideWhenSinglePage` | `boolean` | `false` | Hides the paginator when there is only one page |
| `canGoPrev` | `boolean` | - | Override whether the previous button is enabled |
| `canGoNext` | `boolean` | - | Override whether the next button is enabled |
| `laravel` | `TLaravelPagination \| TLaravelResource` | - | Laravel pagination response for automatic integration |
| `inertiaRouter` | `Router` | - | Inertia.js router for server-side navigation |
| `class` | `string` | - | Additional CSS classes |

## Events

| Event | Payload | Description |
|-------|---------|-------------|
| `change` | `{ page?: number, size?: number, dir?: 'prev' \| 'next' }` | Emitted on any pagination change |
| `update:page` | `number` | Emitted when the current page changes |
| `update:size` | `number` | Emitted when the page size changes |
