# Usage

The `SInputMask` component wraps `SInput` with mask functionality powered by the IMask library. Use the `mask` prop to define the input pattern.

::component-preview{file="SInputMask/basic"}
::

## Mask Patterns

Use different mask patterns to enforce formats like dates, phone numbers, and credit card numbers. The `0` character represents a required digit.

::component-preview{file="SInputMask/masks"}
::

## Error

Set `error` to `true` to display the input with an error style.

::component-preview{file="SInputMask/error"}
::

## Disabled

Set `disabled` to `true` to prevent user interaction.

::component-preview{file="SInputMask/disabled"}
::

## Block

The `SInputMaskBlock` component wraps `SInputMask` with a label, help text, and error text. It combines all the base component props with `label`, `helpText`, and `errorText`. When `errorText` is provided, the error state is automatically applied.

::component-preview{file="SInputMask/block"}
::

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `mask` | `FactoryOpts['mask']` | - | The mask pattern (e.g. `'00/00/0000'`) |
| `modelValue` | `string \| number \| null` | - | The input value (v-model) |
| `placeholder` | `string` | - | Placeholder text |
| `disabled` | `boolean` | `false` | Disables the input |
| `error` | `boolean` | `false` | Displays error styling |
| `rounded` | `'both' \| 'left' \| 'right' \| 'none' \| 'full'` | `'both'` | Border radius style |

The component also accepts all `SInput` props.

## Events

| Event | Payload | Description |
|-------|---------|-------------|
| `update:modelValue` | `string` | Emitted when the masked value changes |
| `complete` | - | Emitted when the mask is fully satisfied |

## Slots

| Name | Description |
|------|-------------|
| `left` | Custom content on the left side of the input |
| `right` | Custom content on the right side of the input |
