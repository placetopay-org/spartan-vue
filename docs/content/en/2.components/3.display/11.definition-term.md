# Usage

The `SDefinitionTerm` component renders a semantic `<dt>`/`<dd>` pair. Provide the term via the `labels` prop (or the default slot) and the content via the `description` prop (or the `description` slot).

::component-preview{file="SDefinitionTerm/basic"}
::

## Multiple Labels

Pass an array of strings to `labels` to render several `<dt>` elements sharing the same description — useful when a single value is known by multiple names.

::component-preview{file="SDefinitionTerm/multipleLabels"}
::

## Using Slots

Use the `default` slot for the label and the `description` slot when the content is not plain text (links, formatted text, icons, etc.). Slots take precedence over the matching props.

::component-preview{file="SDefinitionTerm/slots"}
::

For multiple labels via slots, name them with numeric keys (`#1`, `#2`, …); each becomes its own `<dt>` element:

```vue
<template>
  <SDefinitionTerm>
    <template #1>Phone</template>
    <template #2>Mobile</template>
    <template #description>+57 300 123 4567</template>
  </SDefinitionTerm>
</template>
```

## Oneline (Grouped Layout)

Enable `oneline` to drop the wrapping `<div>` so several `SDefinitionTerm` instances can share a single parent `<dl>`, producing a clean stacked summary.

::component-preview{file="SDefinitionTerm/oneline"}
::

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `labels` | `string \| string[]` | - | The label text rendered as `<dt>`. Pass an array to render multiple terms. Ignored when the `default` or numbered slots are used. |
| `description` | `string` | - | The description text rendered as `<dd>`. Ignored when the `description` slot is used. |
| `oneline` | `boolean` | `false` | Removes the wrapping `<div>` so `<dt>`/`<dd>` render directly in the parent — useful when grouping several instances inside a shared `<dl>`. |
| `class` | `string` | - | Additional CSS classes applied to the root element. |

## Slots

| Name | Description |
|------|-------------|
| `default` | Custom content for the single `<dt>` label. Takes precedence over `labels`. |
| `1`, `2`, `3`, … | Numeric-named slots; each renders as its own `<dt>` element. Used for multiple labels. |
| `description` | Custom content for the `<dd>` description. Takes precedence over `description`. |

## Passthrough

| Key | Element | Description |
|-----|---------|-------------|
| `dt` | `<dt>` | Term label element(s) |
| `dd` | `<dd>` | Description element |

```vue
<template>
  <SDefinitionTerm
    labels="Full name"
    description="John Doe"
    pt:dt="uppercase tracking-wide"
    pt:dd:class="font-semibold"
  />
</template>
```
