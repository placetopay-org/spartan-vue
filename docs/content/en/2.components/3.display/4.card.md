# Usage

The `SCard` component provides a flexible container for content. It supports titles, descriptions, icons, action buttons, and a closable state.

::component-preview{file="SCard/basic"}
::

## Sizes

Use the `size` prop to change the card's padding and border radius.

::component-preview{file="SCard/sizes"}
::

## With Title and Description

Use the `title` prop or the `title` slot, and the `description` slot for additional text.

::component-preview{file="SCard/with-title"}
::

## With Icon

Use the `icon` prop with a status preset (`'success'`, `'danger'`, `'warning'`, `'info'`) to display a predefined glyph with a matching color palette. For non-status visuals, pass a custom Vue component to `icon` and use `iconColor` to set the palette â€” `'primary'` and `'secondary'` are color presets only and live on `iconColor`, not `icon`.

::component-preview{file="SCard/with-icon"}
::

## Icon Type

Use the `iconType` prop to switch between `'solid'` (default) and `'ping'` styles.

::component-preview{file="SCard/icon-type"}
::

## Ping Style

The `'ping'` icon type renders the glyph inside a soft halo of concentric rings, useful for status notifications, system updates, or any card that should grab attention without being intrusive. The ring count adapts to the card `size` (`md`: 4 rings, `sm`: 3 rings) and the alpha of each ring fades from inner (most visible) to outer (almost invisible).

::component-preview{file="SCard/icon-ping"}
::

### Variants

All six color palettes are available with `ping`. The four status presets (`success`, `danger`, `warning`, `info`) are passed directly to `icon`. The `primary` and `secondary` colors apply via `iconColor` paired with a custom glyph.

::component-preview{file="SCard/icon-ping-variants"}
::

## Icon Color Override

Use the `iconColor` prop to override the icon's background, foreground, and ping ring palette independently of the chosen `icon`. `iconColor` accepts the full six-color palette (`'primary'`, `'secondary'`, `'success'`, `'danger'`, `'warning'`, `'info'`) and is the only way to apply `'primary'` or `'secondary'` to a card's icon â€” those are colors, not glyphs, so they are not valid values for `icon` itself.

::component-preview{file="SCard/icon-color"}
::

## Custom Icon

Pass any Vue component to the `icon` prop to use a custom glyph. Combine with `iconColor` to apply any of the six color presets â€” including `'primary'` and `'secondary'`, which are not valid as `icon` strings.

::component-preview{file="SCard/custom-icon"}
::

## Closable

Set `closable` to `true` to display a close button. The component emits a `close` event when clicked.

::component-preview{file="SCard/closable"}
::

## Actions Slot

Use the `actions` slot to add custom action buttons at the bottom of the card.

```vue
<template>
  <SCard icon="success" title="Invitation Sent">
    <template #description>The invitation has been sent to the user.</template>
    <template #actions>
      <SButton variant="primary">Accept</SButton>
      <SButton variant="secondary">Decline</SButton>
    </template>
  </SCard>
</template>
```

## Actions Prop

Use the `actions` prop to render action buttons with icons in a divided footer section.

```vue
<template>
  <SCard
    title="Quick Actions"
    :actions="[
      { icon: PencilIcon, text: 'Edit', onClick: () => console.log('Edit') },
      { icon: TrashIcon, text: 'Delete', onClick: () => console.log('Delete') },
    ]"
  >
    <template #description>Choose an action below.</template>
  </SCard>
</template>

<script setup lang="ts">
import { PencilIcon, TrashIcon } from '@heroicons/vue/24/outline';
</script>
```

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `title` | `string` | - | Card title text |
| `size` | `'sm' \| 'md'` | `'md'` | Card size (affects padding and border radius) |
| `icon` | `FunctionalComponent \| 'success' \| 'danger' \| 'warning' \| 'info'` | - | Icon glyph. Status presets ship with a matching default glyph + color; pass a Vue component for any other glyph |
| `iconColor` | `'primary' \| 'secondary' \| 'success' \| 'danger' \| 'warning' \| 'info'` | - | Overrides the color palette (background, foreground, ping rings). Only way to apply `primary` / `secondary` to an icon |
| `iconType` | `'solid' \| 'ping'` | `'solid'` | Icon display style |
| `closable` | `boolean` | - | Shows a close button in the header |
| `actions` | `TAction[]` | - | Array of action buttons rendered in a divided footer |
| `class` | `string` | - | Additional CSS classes |

## Events

| Event | Payload | Description |
|-------|---------|-------------|
| `close` | - | Emitted when the close button is clicked |

## Slots

| Name | Description |
|------|-------------|
| `default` | Main card content |
| `title` | Custom title content (overrides `title` prop) |
| `description` | Description text below the title |
| `actions` | Action buttons section at the bottom |
