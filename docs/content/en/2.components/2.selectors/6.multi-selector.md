# Usage

The `SMultiSelector` component renders a multi-select dropdown. Use `v-model` to bind the selected values as an array. Pass an array of options via the `options` prop. Selected values are displayed as badges inside the button.

::component-preview{file="SMultiSelector/basic"}
::

## Search

Enable the `search` prop to allow filtering options by text. Use the `query` event to handle filtering externally.

::component-preview{file="SMultiSelector/search"}
::

## Clearable

The `clearable` prop adds a clear button that resets all selected values when clicked.

::component-preview{file="SMultiSelector/clearable"}
::

## Removable

The `removable` prop adds a remove button to each badge, allowing individual items to be deselected directly from the button area.

::component-preview{file="SMultiSelector/removable"}
::

## Compact

The `compact` prop replaces the badges with a text summary showing the number of selections (e.g. "3 selections").

::component-preview{file="SMultiSelector/compact"}
::

## Count

The `count` prop limits the number of visible badges. When more items are selected, a "+N" badge is shown. Defaults to `3`.

::component-preview{file="SMultiSelector/count"}
::

## Badge List

The `badgeList` prop displays a removable badge list inside the dropdown header, allowing users to see and manage all selections within the dropdown area.

::component-preview{file="SMultiSelector/badge-list"}
::

## Rounded

The `rounded` prop controls the border radius of the selector button.

::component-preview{file="SMultiSelector/rounded"}
::

## Disabled

The `disabled` prop prevents user interaction with the selector.

::component-preview{file="SMultiSelector/disabled"}
::

## Error

The `error` prop applies error styling to indicate a validation failure.

::component-preview{file="SMultiSelector/error"}
::

## Loading

The `loading` prop displays a loading indicator inside the selector.

::component-preview{file="SMultiSelector/loading"}
::

## Object Options

When options are objects, use `optionLabel` to specify the display property and `optionValue` to specify the value property used for comparison.

::component-preview{file="SMultiSelector/object-options"}
::

## Grouped Options

Use `optionGroupLabel` and `optionGroupItems` to render options organized in groups.

::component-preview{file="SMultiSelector/grouped-options"}
::

## Custom Option

Use the `option` slot to customize how each option is rendered in the dropdown.

::component-preview{file="SMultiSelector/custom-option"}

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `modelValue` | `(Record<string, any> \| string)[]` | - | The selected values (use with `v-model`) |
| `options` | `(Record<string, any> \| string)[]` | - | Array of available options |
| `placeholder` | `string` | - | Placeholder text when no option is selected |
| `optionLabel` | `string` | `'label'` | Property name to display for object options |
| `optionValue` | `string` | - | Property name used as the value for object options |
| `optionGroupLabel` | `string` | - | Property name for group labels |
| `optionGroupItems` | `string` | - | Property name for group items array |
| `search` | `boolean` | - | Enables a search input in the dropdown |
| `clearable` | `boolean` | - | Shows a clear button when values are selected |
| `removable` | `boolean` | - | Adds a remove button to each badge |
| `compact` | `boolean` | - | Shows selection count text instead of badges |
| `count` | `number` | `3` | Maximum number of visible badges before showing "+N" |
| `badgeList` | `boolean` | - | Displays a badge list inside the dropdown header |
| `handler` | `boolean` | - | Shows an add button when `search` is enabled |
| `rounded` | `'both' \| 'left' \| 'right' \| 'none' \| 'full'` | `'both'` | Border radius style |
| `disabled` | `boolean` | - | Disables the selector |
| `error` | `boolean` | - | Applies error styling |
| `loading` | `boolean` | - | Shows a loading indicator |

## Events

| Event | Payload | Description |
|-------|---------|-------------|
| `update:modelValue` | `(Record<string, any> \| string)[] \| undefined` | Emitted when the selected values change |
| `query` | `string` | Emitted when the search input value changes |
| `add` | `Record<string, any>` | Emitted when a new option is added via the handler |

## Slots

| Name | Props | Description |
|------|-------|-------------|
| `option` | `{ option }` | Custom rendering for each option in the dropdown |
