# Usage

The `SSelector` component renders a dropdown selector. Use `v-model` to bind the selected value. Pass an array of options via the `options` prop.

::component-preview{file="SSelector/basic"}
::

## Search

Enable the `search` prop to allow filtering options by text. The component performs automatic local filtering by default. Set `search="manual"` to handle filtering externally via the `query` event.

::component-preview{file="SSelector/search"}
::

## Clearable

The `clearable` prop adds a clear button that resets the selected value when clicked.

::component-preview{file="SSelector/clearable"}
::

## Rounded

The `rounded` prop controls the border radius of the selector button.

::component-preview{file="SSelector/rounded"}
::

## Disabled

The `disabled` prop prevents user interaction with the selector.

::component-preview{file="SSelector/disabled"}
::

## Error

The `error` prop applies error styling to indicate a validation failure.

::component-preview{file="SSelector/error"}
::

## Loading

The `loading` prop displays a loading indicator inside the selector.

::component-preview{file="SSelector/loading"}
::

## Object Options

When options are objects, use `optionLabel` to specify the display property and `optionValue` to specify the value property used for comparison.

::component-preview{file="SSelector/object-options"}
::

## Grouped Options

Use `optionGroupLabel` and `optionGroupItems` to render options organized in groups.

::component-preview{file="SSelector/grouped-options"}
::

## Custom Option

Use the `option` slot to customize how each option is rendered in the dropdown.

::component-preview{file="SSelector/custom-option"}
::

## Block

The `SSelectorBlock` component wraps `SSelector` with a label, help text, and error text. It combines all the base component props with `label`, `helpText`, and `errorText`. When `errorText` is provided, the error state is automatically applied.

::component-preview{file="SSelector/block"}
::

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `modelValue` | `Record<string, any> \| string` | - | The selected value (use with `v-model`) |
| `options` | `(Record<string, any> \| string)[]` | - | Array of available options |
| `placeholder` | `string` | - | Placeholder text when no option is selected |
| `optionLabel` | `string` | `'label'` | Property name to display for object options |
| `optionValue` | `string` | - | Property name used as the value for object options |
| `optionGroupLabel` | `string` | - | Property name for group labels |
| `optionGroupItems` | `string` | - | Property name for group items array |
| `search` | `boolean \| 'manual'` | - | Enables search. `true` for local filtering, `'manual'` for external filtering via `query` event |
| `clearable` | `boolean` | - | Shows a clear button when a value is selected |
| `rounded` | `'both' \| 'left' \| 'right' \| 'none' \| 'full'` | `'both'` | Border radius style |
| `disabled` | `boolean` | - | Disables the selector |
| `error` | `boolean` | - | Applies error styling |
| `loading` | `boolean` | - | Shows a loading indicator |

## Events

| Event | Payload | Description |
|-------|---------|-------------|
| `update:modelValue` | `Record<string, any> \| string \| undefined` | Emitted when the selected value changes |
| `query` | `string` | Emitted when the search input value changes |

## Slots

| Name | Props | Description |
|------|-------|-------------|
| `option` | `{ option }` | Custom rendering for each option in the dropdown |
